最近由于公司业务需要，要求前端爬取一些网站的信息。基础架构采用先拉取网站的HTML完整文档（fetch-html），再加以解析（extract）（当然这种架构没办法爬取js渲染的html，但是爬HTML里就有的一些纯文字或图片的网站绰绰有余啦٩(๑❛ᴗ❛๑)۶）。于是就接触到了前端的一个爬虫神器–pupeeteer，采用这个库来获取网站HTML文档。在这里接记录一下它的简单用法。

1.pupeeteer简介

介绍：Puppeteer 是一个 Node 库，它提供了高级 API 来通过 DevTools 协议控制 Chromium 或 Chrome。

Puppeteer API 是分层的，并反映了浏览器的结构。

  • Puppeteer 使用 DevTools Protocol 与浏览器通信。
  • Browser 实例可以拥有多个浏览器上下文。
  • BrowserContext 实例定义了一个浏览会话，可以拥有多个页面。
  • Page 至少有一个 frame ：main frame。可能还有其他由 iframe 创建的 frame 或标签。
  • Frame 至少有一个可执行上下文 - 默认执行上下文 - 在其中执行 frame 的 JavaScript 。一个 Frame 可能有与 extensions 有关联的其他执行上下文。
  • Worker 具有单个执行上下文，便于与 WebWorkers 交互。

缺点：

  1. 若页面存在Iframe则无法爬取到完整HTML文档；

  2. 未添加异常处理；

  3. 未制定反反爬策略。如：频繁访问同一网站会受到反爬技术的制约、机器学习等。

  4. … …

在掉了若干头发后~做了以下优化

  1. 引用puppeteer-page-proxy添加页面代理劫持；
  2. 设置User-Agent，每次访问同一网站开启不同视口尺寸；
  3. 若网站内存在Iframe则遍历其所有Iframe拿到完整HTML；
  4. 每次访问前清除cookie，storage等本机存储信息。

3.结语

由于平常只是做页面，APP之类的，刚开始听闻爬虫需求本打算学一波Python，没想到自己的主打语言都可以用来爬虫，刚接触puppeteer时仿佛发现了新大陆，看到爬取结果后还是开心了许久。自己的漫漫前端路又添上了一块新砖，哈哈哈哈~不过还是提醒自己，代码无边界，爬虫需谨慎。
